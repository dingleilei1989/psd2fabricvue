# PSD海报编辑器 - 部署说明

## 项目概述

PSD海报编辑器是一个基于Vue 3的现代化Web应用，实现了以下核心功能：

### ✅ 已完成功能

1. **PSD文件上传和解析**
   - 支持拖拽上传PSD文件
   - 使用`@webtoon/psd`库解析PSD文件结构
   - 提取图层信息、文本内容和图像数据

2. **可视化画布编辑**
   - 使用Fabric.js渲染可编辑画布
   - 支持图层拖拽、缩放、旋转
   - 文本实时编辑功能

3. **图层管理**
   - 图层面板显示所有图层
   - 图层可见性控制
   - 图层删除和重新排序

4. **导出功能**
   - 导出为高质量PNG图片
   - 导出项目JSON数据

5. **现代化UI**
   - 响应式设计，适配移动端
   - 渐变背景和毛玻璃效果
   - 直观的用户交互体验

## 部署步骤

### 1. 环境准备
```bash
# 确保安装了Node.js 16+
node --version

# 确保安装了npm 8+
npm --version
```

### 2. 项目安装
```bash
# 克隆或下载项目
cd psd-poster-editor

# 安装依赖
npm install
```

### 3. 开发环境运行
```bash
# 启动开发服务器
npm run dev

# 访问 http://localhost:5173
```

## 使用指南

### 基本流程
1. **上传PSD文件**: 点击或拖拽PSD文件到上传区域
2. **等待解析**: 系统自动解析PSD图层结构
3. **编辑海报**: 在画布上编辑图层和文本
4. **导出作品**: 保存为PNG图片或JSON数据

### 高级功能
- **文本编辑**: 选中文本图层后可修改内容、字体、颜色
- **图层管理**: 右侧面板控制图层显示和删除
- **工具切换**: 选择工具和文本工具之间切换
- **响应式**: 自动适配不同屏幕尺寸

## 性能优化

### 已实现优化
1. **文件处理**: 使用ArrayBuffer提高大文件读取性能
2. **图层渲染**: 按需渲染避免性能问题
3. **内存管理**: 及时清理临时对象
4. **响应式**: 画布自适应屏幕尺寸

### 推荐配置
- **最大文件大小**: 100MB
- **推荐浏览器**: Chrome 90+, Firefox 85+, Safari 14+
- **内存要求**: 建议4GB+RAM

## 故障排除

### 常见问题

**Q: PSD文件解析失败**
```
原因：文件格式不支持或文件损坏
解决：确认是有效的.psd文件，尝试重新保存
```

**Q: 图层显示异常**
```
原因：图层数据不完整或浏览器兼容性
解决：检查浏览器控制台错误信息
```

**Q: 内存不足**
```
原因：文件过大或图层过多
解决：减小文件大小或关闭其他应用
```

**Q: 编辑功能无响应**
```
原因：JavaScript错误或性能问题
解决：刷新页面或降低图层数量
```
